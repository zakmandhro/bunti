# 🥟 Bunti Icon Engine

**High-fidelity, zero-dependency tactical iconography.**

Bunti features a surgical icon engine designed for the age of agentic engineering. It solves the "Jagged Border" problem by enforcing a **Width-1 Standard** — every icon Bunti renders occupies exactly one terminal column, in every tier.

## 🧭 The Two-Tier System

| Tier | Names | Size | How it loads |
| --- | --- | --- | --- |
| **Curated** | 88 short names (`rocket`, `branch`, `pr`, ...) | built into core | always available |
| **Full Nerd Fonts** | 10,763 names (`fa-rocket`, `md-robot`, `cod-account`, ...) | ~280 KB map (75 KB gzipped) | opt-in subpath import |

Resolution order for `icon(name)`:

1. **Curated `ICON_MAP`** — short aliases with hand-picked, width-audited single-ASCII fallbacks.
2. **Runtime registry** — everything added via `register()` / `registerAll()`, including the full Nerd Fonts pack. A leading `nf-` prefix is stripped, so `nf-fa-rocket` and `fa-rocket` are equivalent.
3. **Miss** — returns `''` and buffers a one-time dev warning (with a nearest-name suggestion) that flushes to stderr on process exit, never mid-frame.

## 🚀 Usage

### 1. The Curated Set (zero cost)

```typescript
bunti.render(({ icon, text }) => {
  text(icon('rocket')); //  (NF) or R (ASCII)
  text(icon('branch')); //  (NF) or * (ASCII)
});
```

### 2. The Full Nerd Fonts Library (one import)

```typescript
import '@zakmandhro/bunti/icons-full'; // side-effect: installs all 10,763 glyphs

icon('fa-rocket');    // 
icon('md-robot');     // 󰚩
icon('nf-fa-rocket'); // 'nf-' prefix accepted too
```

Prefer an explicit call site? The subpath also exports an idempotent installer plus the raw data:

```typescript
import { installFullIcons, NF_GLYPHS, NF_VERSION } from '@zakmandhro/bunti/icons-full';
import type { IconName } from '@zakmandhro/bunti/icons-full'; // 10,763-member union

installFullIcons();
```

The full map is pinned to **Nerd Fonts v3.4.0** and generated by `bun scripts/gen-icons.ts`. It ships as a separate subpath export on purpose — the core `@zakmandhro/bunti` entry never loads it.

### 3. Custom Icons

```typescript
import { register } from '@zakmandhro/bunti';

register('deploy', '');                      // bare glyph ('*' in ASCII tier)
register('deploy', { nf: '', ascii: '^' });  // with explicit ASCII fallback
```

### 4. Side-by-Side Comparison

```bash
bun demo icons
```

## 💎 Tactical Stability

Bunti assumes **Nerd Font v3** support by default for instant high-fidelity rendering.

To ensure layout stability across all terminals, Bunti implements a **Nerd-Emoji Swap**: multi-byte emojis provided by the user are automatically transformed into their closest Width-1 Nerd Font equivalent *before* measurement or rendering.

1. **Nerd Font (default)**: high-resolution tactical symbols (e.g. `󰡯`, ``).
2. **ASCII (fallback)**: surgical *single-character* fallbacks for legacy TTYs (e.g. `R`, `*`). The curated set is width-audited — switching tiers never shifts layout.

On the ASCII tier, raw emoji in your strings pass through **unchanged** — tier degradation applies to Bunti's curated icons (and the Nerd-Emoji Swap only runs on the Nerd Font tier), not to emoji you print yourself.

### Forcing a tier

- **Env override**: `BUNTI_NF=1` (or `NERD_FONT=1` / `NERD_FONTS=1`) forces the Nerd Font tier on for `detectCapabilities()`; `BUNTI_NF=0` forces the ASCII tier, even in terminals detected as modern.
- **Programmatic**: `await init({ nerdFont: false })` (or `render({ nerdFont: false }, ...)`).

Glyphs from the full pack have no hand-picked fallback, so in the ASCII tier they degrade to a generic `*` (still exactly one column). Codegen validates every generated glyph is a single code point of terminal width 1 — width-2 outliers (like `oct-zap`, which is the U+26A1 emoji) are excluded.

## 📐 Spacing Stability

Traditional TUI libraries often struggle with character alignment, causing borders to overflow or cave in when emojis are present.

**Bunti is different.** By mapping complex emojis to fixed-width tactical icons, Bunti guarantees that your layout math is always absolute. A 30-column box will **always** render at exactly 30 columns, regardless of the icons inside.
